COLOR USAGE RULES
  • Background hierarchy: --black (page) > --dark (cards) > --gray (borders/dividers)

DESIGN PRINCIPLES 08
CORE PRINCIPLES
  • Dark Authority: Dark backgrounds with minimal contrast create gravitas and exclusivity
  • Gold Scarcity: Use accent color sparingly - only for CTAs, key highlights, and earned rewards
  • Monospace = System: JetBrains Mono signals "official", "technical", "authoritative"
  • Serif = Human: EB Garamond signals "wisdom", "narrative", "personal"
  • Breathing Room: Generous padding and spacing creates luxury feel (poverty is crowded)
  • Progressive Disclosure: Reveal information gradually, reward attention and effort
  • Subtle Animation: Movement should be barely noticeable - mystery lives in the peripheral
  • Borders as Hierarchy: 1px borders define space without demanding attention
DO NOT
  • Use pure white (#fff) - it's too harsh against dark backgrounds
  • Overuse gold accent - dilutes its power as a reward signal
  • Mix font families randomly - maintain clear semantic roles
  • Use rounded corners - sharp edges maintain the classified aesthetic
  • Add shadows to text - let contrast do the work
  • Use bright colors beyond the defined palette
  • Center-align body text - left-align for readability
  • Use animations for decoration - only for psychological purpose
UX AUDIT // CRITICAL REVIEW 09
The following observations are from a UX/UI expert perspective. These are suggestions for v2.0, not requirements.
ACCESSIBILITY CONCERNS // PRIORITY: HIGH
  • Minimum Font Size: Sizes below 0.7rem (11.2px) fail WCAG readability guidelines. The 0.55rem and 0.6rem specimens are problematic for users with visual impairments. Suggestion: Consolidate to minimum 0.7rem for any readable text.
  • Color Contrast: --dim (#888) on --black (#0a0a0a) yields only 5.3:1 ratio. Acceptable for large text but fails for small labels. Suggestion: Use #999 or higher for small dim text.
  • Focus States Missing: No documented keyboard focus indicators. Users navigating via Tab key have no visual feedback. Suggestion: Add outline: 2px solid var(--accent) with offset for all interactive elements.
  • Touch Targets: Some buttons have padding of 10px which results in touch targets below 44x44px minimum for mobile. Suggestion: Ensure minimum 44px height on all buttons.
TYPOGRAPHY SCALE // PRIORITY: MEDIUM
  • Too Many Sizes: Current scale has 12+ font sizes. Industry standard recommends 6-8 maximum for consistency. Suggestion: Consolidate to: 6rem, 2.5rem, 1.5rem, 1.1rem, 1rem, 0.85rem, 0.75rem, 0.7rem.
  • Letter Spacing Inconsistency: Five different letter-spacing values (1px, 2px, 3px, 4px, 8px) lack mathematical relationship. Suggestion: Use powers of 2: 1px, 2px, 4px, 8px only.
  • Line Height Variants: Four line-heights (1.6, 1.7, 1.8, 1.9) are too granular to notice. Suggestion: Use only 1.5 (compact), 1.7 (default), 1.9 (relaxed).
  • Missing Type Variables: Font sizes are hardcoded rather than using CSS custom properties. Suggestion: Create --font-size-xs through --font-size-4xl variables.
SPACING SYSTEM // PRIORITY: MEDIUM
  • Non-Mathematical Scale: Current: 5, 10, 15, 20, 25, 30, 40, 50. This is neither base-4 nor base-8. Suggestion: Adopt 4px base: 4, 8, 12, 16, 24, 32, 48, 64, 96.
  • Inconsistent Jumps: Scale jumps from +5 increments to +10 (30>40>50) without reason. Creates unpredictable visual rhythm.
  • No Negative Space: No documentation for margin-auto centering, flex gaps, or grid gutters.
MISSING COMPONENTS // PRIORITY: MEDIUM
  • Loading States: No skeleton screens, spinners, or progress indicators documented. The experience uses loading moments.
  • Form Validation: Only error state shown. Missing: success checkmark, warning, info, loading states for inputs.
  • Audio Player: The morse.wav player exists in landing but not documented in library. Inconsistent with other components.
  • Modal/Overlay: Contract modal exists but no reusable modal pattern documented.
  • Tooltips: No hover explanation patterns for complex UI elements.
  • Responsive Breakpoints: No mobile/tablet/desktop breakpoint documentation. Only one @media query at 800px.
COLOR SEMANTICS // PRIORITY: LOW
  • Gold Overload: --accent (#d4af37) is used for: CTAs, success, selected, verified, highlights, links, labels. Too many semantic meanings dilute its significance. Suggestion: Consider a secondary accent for non-CTA highlights.
  • No Warning State: Only error (red) exists. No amber/orange for non-critical warnings. The forfeit confirmation uses red but isn't an error.
  • Error Variants Overkill: Four error colors (--error, --error-light, --error-muted, --error-border) may be excessive. Suggestion: Reduce to --error and --error-subtle.
BUTTON HIERARCHY // PRIORITY: LOW
  • Primary vs CTA Confusion: Both are gold-themed. Primary is outline, CTA is filled. User may not understand hierarchy difference.
  • Danger Button Too Subtle: Red outline on dark background is less visible than gold. Destructive actions should be MORE prominent, not less.
  • No Ghost Button: Missing a minimal button style for tertiary actions (e.g., "Cancel" that shouldn't compete with primary).
WHAT'S WORKING WELL // STRENGTHS
  • Strong Visual Identity: The dark + gold palette is distinctive, memorable, and creates genuine premium feel.
  • Clear Font Pairing: Serif for humanity, mono for system. The semantic separation is clear and consistent.
  • Psychological Coherence: Every element serves the "classified/exclusive" narrative. No aesthetic compromises.
  • Animation Restraint: Subtle breathe/pulse animations create life without distraction. Professional-grade.
  • Information Hierarchy: Clear visual hierarchy guides user attention effectively through complex flows.
  • Iconography: Unicode symbols over custom icons is pragmatic and ensures cross-platform consistency.
